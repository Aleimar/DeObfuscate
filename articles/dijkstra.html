Dijkstra's Algorithm

To understand the concept of Dijkstra’s algorithm, we need to first familiarize ourselves with some foundational information. Let’s begin by discussing graphs.

             A graph is a visual representation of data, organized in a way that shows relationships between different elements. It is created using a set of points, known as vertices, which are connected by lines or arcs, known as edges. The graph provides a clear way to see the connection between the different elements, making it easier to understand the relationships.

Graphs come in two forms: directed and undirected. A directed graph is made up of a set of vertices (nodes) connected by directed edges, meaning the edges have a specific direction. On the other hand, an undirected graph consists of a set of nodes connected by bidirectional edges, meaning the edges don’t have a specific direction. These graphs are sometimes referred to as undirected networks.

To continue, we will focus on undirected graphs in the upcoming examples. Additionally, there is a type of graph known as a weighted graph, which is characterized by assigning a numerical value, typically a positive number, to each edge.

Let’s now discuss Dijkstra’s algorithm:

Dijkstra’s algorithm is a tool used to find the shortest route between two points in a graph. Imagine a map with many roads connecting different cities. If you want to travel from one city to another, Dijkstra’s algorithm will help you find the quickest way to get there. The algorithm considers the distance between each city, represented by lines on the map, and uses this information to determine the fastest route. Created by Dutch computer scientist Edgar W. Dijkstra, the algorithm only works with graphs that have non-negative weights, meaning the distances between cities are always positive numbers. The purpose of the algorithm is to solve the shortest-path problem by finding the most efficient route from the starting node to every other node in the graph.

What is the shortest-path problem?

The shortest path problem involves finding the most efficient route between two points in a network, such as roads or connections. This problem is important in various industries, including transportation, communications, and geography. To solve this problem, Dijktra’s algorithm, A* algorithm, or Bellman-Ford algorithm is used.

How it works - A simplified version

    1. Initialize two sets of vertices, one for unvisited and one for visited vertices. Assign a distance value to each vertex. The starting vertex will have a distance value of 0, while all other vertices will have a distance value of infinity.
    2. Select the unvisited vertex with the smallest distance value and label it as the current vertex.
    3. For each vertex neighboring the current one, calculate the distance from the current vertex to that neighbor. If this distance is shorter than the current distance value for that neighbor, update the distance value.
    4. Mark the current vertex as visited and transfer it to the set of visited vertices.
    5. Keep repeating steps 2 to 4 until all vertices have been visited or a predetermined stopping point has been reached.
    6. The distance values for each vertex show the shortest distance from the starting point to that vertex. The actual route can be determined by tracing the path with the smallest distance from the starting vertex to the desired endpoint.

Conclusion:

Dijkstra's algorithm, while useful for determining the shortest path between multiple nodes, may not perform optimally when trying to find the shortest route to a specific destination. The algorithm operates by always choosing the shortest path from the unvisited list, without taking into account the target's location or proximity.
