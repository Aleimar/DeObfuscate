The structure of a Compiler

Computer programming is the process of writing a set of instructions to tell a computer how to execute a task. Programming languages such as C++, Java, and Python are high-level languages designed to be easily understood by humans to create different computer programs. But machines only executes instructions written in machine-readable code also called low-level languages which directly translate to registers, memory locations, and other computer components.

             So how does computers execute high-level language written by programmers if it could only execute low-level instructions?

             Answer: COMPILERS.

So how does the compiler do it?

The compiler do this in two phases. Analysis and Systhesis.

Analysis
Synthesis
Breaking up of source program into pieces for intermediate representation. Construction of target program or machine level code.

Note: The following phases are written to give an overview of each compiler phases. For deeper understanding of each phases, we wrote specific articles for them which you may read through the links below.

LEXICAL ANALYSIS

The first step is to recognize each words in a statement.

Lexical analysis (also called scanning), reads streams of character from source program and group these characters into meaningful sequences called tokens.

Consider the following statement.

Alex love her dog.

In the English language, we group letters to form words before we read it. In the statement above, we have the words "Alex","love","her", and "dog".

Now, let's look into this programming statement:

force = mass * 9.8.

In programming, "force","=", "mass", "*", and "9.8" are called lexemes. Each lexeme will be processed to become a lexical token (or token), a group of characters with identified meaning. You can read more about Lexical Analysis here.

SYNTAX ANALYSIS

The next step is to understand the statement.

Syntax analysis (or parsing), uses tokens from lexical analysis to create a tree-like intermediate representation that depicts the grammatical structure also known as syntax tree.

SEMANTIC ANALYSIS

Then we analyze the statement.

After the syntax tree is generated, compilers check whether the source program is consistent with semantic rule. It checks whether the statement has a clear meaning and consistent with what they're supposed to be used.

Alex love her dog.

In English language, the statement above could be sematically analyzed through the following questions:

1. What did subject love?

If the statement is just "Alex love.", is it still consistent with the rules of English language?

2. Is there a subject in the statement?

Now if the statement is just "Love her dog.", will it still hold the same meaning from the original statement?

 

force = mass * 9.8

In programming language, the statement above could be sematically analyzed through the following questions:

1. For multiplication operation, does it have value for multiplicand and multiplier?

2. Does the assignment statement have clear assigned values?

Semantic Analysis also checks data types, wrong arguments, undeclared/multiple declarations, etc. More of this can be read through this article.

CODE OPTIMIZATION

Then we edit.

This is a transformational stage where we improve code by minimizing reading time, use less memory and other resources.

Alex love dogs.

From the original statement, "Alex love her dog.", we can use the statement above instead (assuming its true) to minimize reading time but still having the same meaning and usage.

mass = 0

force = mass * 9.8

Now, if the source program defined the value of mass as 0, the compiler can then optimize and change the statement to:

force = 0

Annihilating the multiplication statement reduces memory usage, and time computation.

You can read more about Code Optimization here.

CODE GENERATION

Finally, machine translation.

This is the last stage of compiler where actual code will be translated to machine instruction.

force = 0

Let's say R1 means register (a type of computer memory), LD is a load instruction, ST is a store instruction, then we can translate the statement above to:

1. R1 LD, force

2. ST R1, 0

The instructions above will be read by the computer as:

1. load (assign) the force to Register Address 1.

2. Store the value '0' to Register Address 1.

Note: The example above is just an overview of what is the Code Generation phase for compilers. Different machines have different set of instructions. And for deeper understanding of each phases, we wrote specific articles for them which you may read through the links below.
